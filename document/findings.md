# Project Findings: Desktop Pet & Toolkit App

## 1. Project Overview
打造一款 Android 应用，核心包含一个可交互的 2D 桌宠助手，并为未来的工具箱功能预留入口。项目采用 MVP (Minimum Viable Product) 策略，优先保证桌宠交互体验。

## 2. Goals & Non-Goals (范围界定)

### Goals (必须实现)
- **平台**: Android 原生应用。
- **桌宠**:
  - 2D 平面形象。
  - **范围限制**: 仅在 App 内部活动，不申请系统级悬浮窗权限。
  - **基础交互**: 待机动画、点击反馈、拖拽移动。
  - **高级交互**: 接入 LLM (大语言模型) 进行对话。
- **工具箱**: 在 UI 上预留入口。

### Non-Goals (本阶段不做)
- **系统级悬浮**: 桌宠不需要在 App 退到后台时显示。
- **工具箱逻辑**: 点击工具箱按钮仅提示“开发中”，不开发具体工具功能。
- **复杂动画**: 不包含复杂的 3D 模型或骨骼动画，基于帧动画或静态图切换即可。

## 3. Key User Flows (核心流程)
1. **启动**: App 启动 -> 桌宠默认出现在屏幕**右下角** -> 顶部导航栏显示“工具箱”按钮。
2. **移动**: 用户**长按**桌宠 -> 震动反馈 -> 拖动手指 -> 桌宠跟随移动。
3. **对话**: 用户**双击**桌宠 -> 弹出键盘和输入框 -> 对话气泡显示在桌宠**左侧**。
4. **工具**: 点击顶部“工具箱” -> 弹出 Toast 提示“开发中”。

## 4. Risk Mitigation (风险与决策)
- **手势冲突**: 采用 **"单击(互动) / 双击(对话) / 长按(拖拽)"** 的区分方式，避免操作混淆。
- **键盘遮挡**: 接受键盘弹起时可能暂时遮挡桌宠的情况，侧重于保证输入框可见。
- **LLM 延迟**: 在等待 LLM 回复期间，桌宠保持**静止/待机状态**，不增加额外的 Loading 动画复杂度。
